# Utilities Reference

Internal utility functions used by swifty-docs build pipeline and theme components. Some utilities are exported for advanced use cases, while others are used internally by the scanner, compiler, and theme views.

## Text Processing

### slugify

Convert heading text into a URL-safe slug string for anchor links.

```typescript
function slugify(text: string): string;
```

Parameters:

- `text` - Heading text to convert

Returns a URL-safe slug string.

Conversion rules:

- Lowercase the text
- Preserve CJK, Cyrillic, Arabic, and other non-ASCII scripts using Unicode property escapes (`\p{L}` for letters, `\p{N}` for numbers)
- Replace non-letter/number/space/dash characters with a dash (preserves word boundaries: `"Hello!World"` becomes `"hello-world"`, not `"helloworld"`)
- Replace whitespace sequences with a single dash
- Collapse consecutive dashes
- Trim leading and trailing dashes
- Prefix a leading digit with underscore so the slug is a valid CSS selector (`querySelector("#123")` is invalid; `"#_123"` is valid)

Example:

```typescript
import { slugify } from "swifty-docs";

slugify("Hello World"); // "hello-world"
slugify("路由导航"); // "路由导航"
slugify("What's New?!"); // "what-s-new"
slugify("v2.0 Changes"); // "v2-0-changes"
slugify("123 Release"); // "_123-release"
slugify("already-slugged"); // "already-slugged"
slugify("a -- b"); // "a-b"
```

Exported from `swifty-docs` runtime entry point.

### deriveTitleFromPath

Derive a human-readable title from a file path.

```typescript
function deriveTitleFromPath(filePath: string): string;
```

Parameters:

- `filePath` - File path relative to the docs directory (e.g., `"guide/config.md"`, `"index.md"`)

Returns a formatted title string.

Behavior:

- For `index.md` files, uses the parent directory name with dashes replaced by spaces and words capitalized (e.g., `"guide/index.md"` becomes `"Guide"`)
- Falls back to `"Home"` when `index.md` has no parent directory (root level)
- For other files, replaces dashes with spaces and capitalizes words (e.g., `"getting-started.md"` becomes `"Getting Started"`)

Example:

```typescript
import { deriveTitleFromPath } from "swifty-docs";

deriveTitleFromPath("index.md"); // "Home"
deriveTitleFromPath("guide/index.md"); // "Guide"
deriveTitleFromPath("api/index.md"); // "Api"
deriveTitleFromPath("getting-started.md"); // "Getting Started"
deriveTitleFromPath("quick-start.md"); // "Quick Start"
```

Internal utility used by the scanner and compiler to generate page titles when frontmatter does not provide one. Not available via any public import path.

## Markdown Extraction

### extractFirstHeading

Extract the first h1 heading text from markdown content.

```typescript
function extractFirstHeading(content: string): string | undefined;
```

Parameters:

- `content` - Raw markdown content

Returns the plain text of the first h1 heading, or `undefined` when no h1 exists.

Behavior:

- Uses markdown-it to parse content into tokens
- Walks the token stream to find the first `heading_open` token with tag `"h1"`
- Collects plain text from the inline token's children (bold, italic, and link markers are stripped)
- Code blocks are ignored because they produce `code_block` tokens, not `heading_open` tokens

Example:

```typescript
import { extractFirstHeading } from "swifty-docs";

const content = `
# Introduction

Some text here.

## Getting Started

More text.
`;

extractFirstHeading(content); // "Introduction"
```

Internal utility used by the scanner to extract page titles when frontmatter does not provide one. Not available via any public import path.

### extractHeadings

Extract h2 and h3 headings from markdown content for table of contents generation.

```typescript
function extractHeadings(content: string): HeadingInfo[];

interface HeadingInfo {
  level: number;
  text: string;
  slug: string;
}
```

Parameters:

- `content` - Raw markdown content

Returns an array of `HeadingInfo` objects, each containing:

- `level` - Heading level (2 for h2, 3 for h3)
- `text` - Plain text content (inline markers stripped)
- `slug` - URL-safe slug generated by `slugify()`

Behavior:

- Parses content with markdown-it and walks the token stream
- Collects only h2 and h3 headings (h1 and h4+ are ignored)
- Skips headings with empty text content
- Generates slugs using the same `slugify()` function used by the anchor plugin, ensuring consistency between TOC links and heading anchors

Example:

```typescript
import { extractHeadings } from "swifty-docs";

const content = `
# Page Title

## Introduction

Some text.

### Getting Started

More text.

## Configuration

Final section.
`;

extractHeadings(content);
// [
//   { level: 2, text: "Introduction", slug: "introduction" },
//   { level: 3, text: "Getting Started", slug: "getting-started" },
//   { level: 2, text: "Configuration", slug: "configuration" }
// ]
```

Internal utility used by the scanner to populate `pageData.headings` for TOC rendering. Not available via any public import path.

### extractExcerpt

Extract a plain-text excerpt from markdown content for search indexing.

```typescript
function extractExcerpt(content: string, maxLen?: number): string;
```

Parameters:

- `content` - Raw markdown content
- `maxLen` - Maximum length of the excerpt in characters (default: 200)

Returns a plain-text string with heading text excluded and whitespace collapsed.

Behavior:

- Parses content with markdown-it and collects text from inline tokens
- Skips inline tokens that immediately follow a `heading_open` token (excludes heading text from excerpts)
- Concatenates text from body content only
- Collapses multiple whitespace characters into single spaces
- Trims leading and trailing whitespace
- Truncates to `maxLen` characters

Example:

```typescript
import { extractExcerpt } from "swifty-docs";

const content = `
# Page Title

This is the first paragraph with some **bold** text.

## Introduction

Second paragraph with more content.
`;

extractExcerpt(content);
// "This is the first paragraph with some bold text. Second paragraph with more content."
```

Internal utility used by the scanner to populate `pageData.excerpt` for search indexing. Not available via any public import path.

## Route Utilities

### sortDocsRoutes

Sort an array of documentation routes using the "all or nothing" sidebar position rule.

```typescript
function sortDocsRoutes(routes: DocsRoute[]): void;
```

Parameters:

- `routes` - Array of `DocsRoute` objects to sort in place

Returns `void`. The array is sorted in place.

Behavior:

- Checks whether all routes in the array have `pageData.sidebarPosition` defined
- If all routes have `sidebarPosition`: sort by position ascending, then by filename (dictionary order) as tiebreaker
- If any route is missing `sidebarPosition`: ignore all positions and sort by filename only (dictionary order)
- Position values are 0-based (0 = first, 1 = second, etc.)

Example:

```typescript
import { sortDocsRoutes } from "swifty-docs";

const routes = [
  {
    path: "/docs/guide/z-index",
    viewId: "guide-z-index",
    filePath: "/absolute/path/to/guide/z-index.md",
    pageData: { sidebarPosition: 2, relativePath: "guide/z-index.md" },
  },
  {
    path: "/docs/guide/config",
    viewId: "guide-config",
    filePath: "/absolute/path/to/guide/config.md",
    pageData: { sidebarPosition: 0, relativePath: "guide/config.md" },
  },
  {
    path: "/docs/guide/intro",
    viewId: "guide-intro",
    filePath: "/absolute/path/to/guide/intro.md",
    pageData: { sidebarPosition: 1, relativePath: "guide/intro.md" },
  },
];

sortDocsRoutes(routes);
// routes is now sorted: config, intro, z-index
```

Internal utility used by the scanner to determine sidebar order and virtual index route targets. Not available via any public import path.

## DOM Utilities

### findDataHref

Walk up the DOM tree from an event target to find the nearest element carrying a `data-href` attribute.

```typescript
function findDataHref(target: EventTarget | null): string | null;
```

Parameters:

- `target` - Event target (typically from a click event)

Returns the `data-href` value as a string, or `null` if no element with `data-href` is found in the ancestor chain.

Behavior:

- Starts from the event target and walks up through parent elements
- Stops at the first element that has a `data-href` attribute in its dataset
- Returns the attribute value or `null` if the root is reached without finding the attribute

Usage:

Theme templates render anchor elements with `data-href` attributes and click handlers. Clicks may land on child elements (SVG icons, spans, marks), so the handler must walk up to the element that carries the attribute:

```typescript
import { findDataHref } from "swifty-docs";

function handleClick(event: MouseEvent) {
  const href = findDataHref(event.target);
  if (href) {
    Router.to(href);
  }
}
```

Internal utility used by theme views (sidebar, TOC) to handle navigation clicks on elements with nested children. Not available via any public import path.

### getFirstRoute

Get the first route from a sorted array of documentation routes.

```typescript
function getFirstRoute(routes: DocsRoute[]): DocsRoute | undefined;
```

Parameters:

- `routes` - Array of `DocsRoute` objects

Returns the first route from a sorted copy of the array, or `undefined` if the array is empty. The function sorts a copy of the array without mutating the input, using the same "all or nothing" sidebar position rule as `sortDocsRoutes`.

Behavior:

- Creates a shallow copy of the routes array
- Sorts the copy using `sortDocsRoutes` logic
- Returns the first element (or `undefined` for empty arrays)
- Does not modify the original input array

Used by the scanner to select the target page for virtual index routes when a directory contains no `index.md` file. Not available via any public import path.
