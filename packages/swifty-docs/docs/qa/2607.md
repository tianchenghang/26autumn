## QA: CSS

### CSS 选择器的优先级

specificity 特异性四元组 (a, b, c, d)

- a: 内联样式
- b: ID 选择器的数量
- c: 类, 伪类, 属性选择器的数量
- d: 元素, 伪元素选择器 (例如 ::before, ::after) 的数量

\* 通配符选择器, > 子选择器, + 相邻兄弟选择器, ~ 通用兄弟选择器不影响选择器的优先级

建议

- 避免使用 ID 选择性: 特异性高, 导致样式覆盖难
- 避免使用 !important: !important 特异性最高, 破坏层叠规则
- 使用 BEM 命名规范

### Web Component 样式隔离

Web Component 使用 Shadow DOM 实现样式隔离, 宿主元素调用 `el.attachShadow({ mode: 'open' })`, 在宿主元素内挂载一棵影子 DOM 树, 影子 DOM 树有独立的 DOM 作用域和样式作用域

Shadow DOM 可以隔离 CSS 选择器, 不能隔离 CSS 继承

```html
<style>
  :root {
    /* 不能隔离 CSS 继承 */
    --theme-primary: lightblue;
    font-family: "Menlo", sans-serif;
  }
  .card {
    /* 可以隔离 CSS 选择器 */
    color: lightgreen;
  }
</style>

<div class="card">lightgreen text</div>
<my-card></my-card>

<script>
  class MyCard extends HTMLElement {
    constructor() {
      super();
      const shadow = this.attachShadow({ mode: "open" });
      shadow.innerHTML = `
      <style>
        .title {
          color: var(--theme-primary, #000);
        }
        .card2 {
          padding: 1rem;
          border: 1px solid #ccc;
          border-radius: 0.5rem;
        }
      </style>
      <div class="card2">
        <h3 class="title">lightblue text</h3>
        <p>Menlo font, black text</p>
      </div>
    `;
    }
  }
  customElements.define("my-card", MyCard);
</script>
```

### 介绍 CSS 原子化、css-in-js、CSS 模块化

CSS 原子化 (tailwindcss)

- 组合 CSS 原子类, 而不是自定义 CSS
- 构建时扫描源文件, 提取所有使用的类名
- 支持 JIT (Just-In-Time) 按需生成 CSS 规则
- 构建产物是静态 CSS 文件, 零 JS 运行时开销

css-in-js (styled-components)

- 渲染时 (运行时) 将模版字符串解析为 CSS 字符串
- 计算 CSS 字符串的 hash, 生成唯一 CSS 类名
- 通过 `<style>` 或 `CSSStyleSheet.insertRule()` 插入 html 文档
- 有缓存, 一个样式只插入一次

CSS 模块化 (.module.css)

- 构建工具 (Webpack CSS loader、Vite) 处理 .module.css 时, 对 CSS 类名加 hash 后缀
- 构建产物是静态 CSS 文件, 零 JS 运行时开销

性能: CSS 原子化 = CSS 模块化 > css-in-js

CSS 原子化和 CSS 模块化的构建产物都是静态 CSS 文件, 可以 preload, 可以并行解析 DOM 树和 CSSOM 树、可以独立缓存
而 css-in-js 是运行时生成 CSS 样式并插入 html 文档

### PostCSS

CSS 中的 Babel

- 将 CSS 字符串解析为 AST 抽象语法树
- 插件遍历、修改 AST 节点
- 将修改后的 AST 重新序列化为 CSS 字符串
- 支持生成 Source Map

场景

- CSS 语法降级
- autoprefixer 添加浏览器前缀
- CSS 压缩
- px 转 rem/..., 移动端适配
- tailwindcss
- stylelint

### React Native 的 CSS 对比 Web 的 CSS, Yoga 布局引擎

React Native 不是浏览器, 没有 DOM、没有 CSSOM; RN 中的 CSS 是使用 CSS 属性名和 flexbox 语义的 JS 对象, 使用 JS 对象描述样式, 底层的 Yoga 布局引擎计算布局

- RN 没有 css 文件、不支持 CSS 选择器、不支持 CSS 继承/层叠 (cascade)
- RN 数值没有单位, 是逻辑像素, 不支持 em、rem、vw、vh...
- RN 不支持伪类 (:hover)、不支持伪元素 (::before, ::after)
- RN 不支持浮动
- RN 只支持 flex 布局, 默认 `flexDirection: "column"`, Web 默认 `flex-direction: row`
- RN 不支持 CSS 过渡 (transition)、不支持 CSS 动画 (@keyframes)
- RN 不支持 CSS 变量、CSS 函数
- RN 不支持媒体查询 (@media)
- RN 不支持

Yoga 布局引擎

1. RN 旧架构: JS 的 style 对象通过 bridge 异步发送给 Yoga 节点树, 有序列化开销
1. RN 新架构: JS 的 style 对象通过 JSI 同步给 Yoga 节点树, 没有序列化开销
1. Yoga 构建布局树, 每个 RN 组件对应一个 Yoga 节点
1. Yoga 执行 flexbox 布局算法, 计算布局
1. Yoga 将计算结果跨「线程」提交给 iOS/Android 进行渲染

### 三栏布局: 中间宽度固定、两边宽度自适应

方案 1: flex 布局

- 容器: 开启 flex 布局
- 左右元素: `flex: 1`
- 中间元素: `flex: 0 0 500px`

```html
<style>
  .container {
    display: flex;
  }

  .left,
  .right {
    min-width: 0;
    /* 即 flex: 1 1 0%; 可以拉伸, 可以压缩, 主轴方向的初始大小为 0% */
    flex: 1;
    background: lightgreen;
  }

  .center {
    width: 500px;
    /*  不能拉伸, 不能压缩, 主轴方向的初始大小等于 500px */
    flex: 0 0 500px;
    background: lightblue;
  }
</style>

<div class="container">
  <aside class="left">left</aside>
  <main class="center">center 500px</main>
  <aside class="right">right</aside>
</div>
```

方案 2: grid 布局

- 容器: 开启 grid 布局, `grid-template-columns: 1fr 500px 1fr;`

```html
<style>
  .container {
    display: grid;
    grid-template-columns: 1fr 500px 1fr;
  }
</style>

<div class="container">
  <aside style="background: lightgreen;">left</aside>
  <main style="background: lightblue;">center 500px</main>
  <aside style="background: lightgreen;">right</aside>
</div>
```

方案 3: 浮动

- 左、中、右全部左浮动或右浮动
- 中间元素宽度固定 500px, 左右元素宽度使用 `calc((100% - 500px) / 2)` 计算
- 父元素使用 `::after` 创建空的伪元素以清除浮动

```html
<style>
  /* 父元素使用 `::after` 创建空的伪元素以清除浮动 */
  .container::after {
    content: "";
    display: block;
    clear: both;
  }

  .center {
    float: left;
    width: 500px;
    background: lightblue;
  }

  .left {
    float: left;
    width: calc((100% - 500px) / 2);
    background: lightgreen;
  }

  .right {
    float: left;
    width: calc((100% - 500px) / 2);
    background: lightgreen;
  }
</style>

<div class="container">
  <aside class="left">left</aside>
  <main class="center">center 500px</main>
  <aside class="right">right</aside>
</div>
```

### CSS 动画

> CSS 动画的实现方式: transform 变换、transition 过渡、@keyframe 关键帧

- 只能定义起始和结束两个状态, 无法定义中间关键帧
- 需要事件触发 (hover、class 切换等), 不能自动播放
- 适合简单的状态切换动画, 如按钮悬停、展开收起
- 性能较好, 可动画属性中 transform 和 opacity 走合成层

- 可以定义多个关键帧, 实现复杂动画序列
- 可以自动播放、循环、反向播放
- 适合加载动画、轮播、复杂序列动画
- 支持暂停和恢复

- 不是独立的动画方式, 通常配合 transition 或 animation 使用
- 不触发布局回流, 性能最优
- 可以开启 GPU 加速 (合成层)

| transition 过渡            | animation 动画               | transform 变换 |
| -------------------------- | ---------------------------- | -------------- |
| 仅有开始和结束两个状态     | 可以有多个关键帧             | 不适用         |
| 需要事件触发, 例如鼠标悬浮 | 不需要事件触发, 可以自动播放 | 不适用         |
| 仅触发 1 次                | 可以循环多次或无限循环       | 不适用         |
| 性能较好                   | 性能较好                     | 性能最好       |
| 状态切换                   | 复杂动画                     | 位移/旋转/缩放 |

| 对比           | transform 变换    | position 定位                              |
| -------------- | ----------------- | ------------------------------------------ |
| 是否脱离文档流 | 不脱离文档流      | absolute 绝对和 fixed 固定定位会脱离文档流 |
| 性能           | 高, 可以 GPU 加速 | 低, 会触发 reflow 回流                     |
| 百分比参考     | 元素自身尺寸      | 父元素尺寸                                 |

### CSS GPU 加速

浏览器渲染一帧的画面

1. 执行 JS
2. 计算样式
3. 回流 reflow, 有关宽高等, 性能开销大
4. 重绘 repaint, 有关颜色等, 性能开销小
5. 合成 composite: 将多个图层 layer 合并为渲染的画面

GPU 加速: 对于 transform 变换, opacity 不透明度, 可以跳过回流和重绘, 直接进入合成阶段使用 GPU 将多个图层 layer 合并为渲染的画面

触发 GPU 加速

- transform 变换, opacity 不透明度
- 提示浏览器 `will-change: transform;`, `will-change: opacity;`

### `@layer` 显式声明样式的优先级层级

- `@layer` 显式声明样式的优先级层级
- 后声明的 layer 的样式优先级更高
- 不属于任何 layer 的样式优先级最高
- 解决第三方库样式覆盖困难, 特异性战争等问题

```css
/* 后声明的 layer 优先级更高 */
@layer reset, base, components;

@layer reset {
  * {
    margin: 0;
    padding: 0;
  }
}

@layer base {
  .input:focus,
  .input:focus-within,
  .textarea:focus,
  .textarea:focus-within {
    outline: none;
  }
}
```

### `<link>` 和 `@import`

- `<link>` 是 html 标签, `@import` 是 CSS 规则
- 多个 `<link>` 可以并行下载, 浏览器预加载扫描器 (preload scanner) 可以在解析 html 文档的早期发现 `<link>`, `<script>`, `<img>` 并请求; `@import` 的 CSS 必须等到父 CSS 下载并解析后, 才能被请求, 形成串行瀑布
- `<link>` 支持 `rel="preload"` 预加载

### CSS 的阻塞

- CSS 不会阻塞 DOM 树的构建
- CSS 会阻塞 DOM 树的渲染: 等待 CSSOM 树构建完成后, 才能将 DOM 树和 CSSOM 树合并为渲染树 (Render Tree)
- CSS 会阻塞后续 JS 脚本的执行: 才能执行后续的 JS 脚本

### CSS 性能优化

1. 关键 CSS 内联到 html
2. 非关键 CSS: `rel: "preload stylesheet" as="style"` 预加载、as 指定资源类型, `onload="this.rel='stylesheet'"` CSS 下载完成后, 通过 onload 将 rel 从 preload 改为 stylesheet
3. 非关键 CSS: `media="print"` 浏览器匹配媒体查询时, 如果发现 CSS 的 `media="print"` 和当前环境不匹配 (例如 `media="screen"`), 则将该 CSS 视为非阻塞资源, 不会先等待该 CSS 下载完成后, 再渲染页面; `onload="this.media='all'"` 该 CSS 下载完成后, 通过 onload 将 media 从 print 改为 all
4. 按路由/页面拆分 CSS chunk

### CSS 的选择器从右向左匹配

CSS 的选择器从右向左匹配, 最右侧的选择器称为关键选择器 (key selector)

blink 渲染引擎匹配样式时, 先使用最右侧的关键选择器在 DOM 树中筛选出候选元素集合, 每个候选元素沿 DOM 树向上, 验证左侧的选择器

关键选择器的选择性要高

### content-visibility

- `content-visibility: visible`: 默认, 正常渲染
- `content-visibility: hidden`: 有宽高, 不响应事件, 触发回流、重绘
- `content-visibility: auto`: 视口外的元素跳过渲染

## 隐藏元素

| 隐藏方式             | 是否有宽高               | 是否响应事件 | 回流/重绘  |
| -------------------- | ------------------------ | ------------ | ---------- |
| `display: none`      | 否, 宽高 = 0, DOM 未卸载 | 否           | 回流、重绘 |
| `visibility: hidden` | 是                       | 否           | 重绘       |
| `opacity: 0`         | 是                       | 是           | 重绘       |

`content-visibility: auto` 实现虚拟滚动列表: 视口外的元素跳过渲染, 降低浏览器的渲染成本

例如渲染 10000 个 item 的大列表, 仍然创建 10000 个虚拟 DOM、仍然挂载 10000 个 DOM 节点, 只是视口外的 item 跳过渲染

对比 `@tanstack/react-virtual`, 如果可视区可以放 10 个 item, 则只创建 10 + n 个虚拟 DOM, 只挂载 10 + n 个 DOM 节点, 减小 DOM 节点数量 (n 是预挂载的 item 数量)

减小 DOM 节点数量: 降低框架 diff、DOM 变更、样式计算、布局、绘制的开销, 降低内存占用

### new 一个对象

`const obj = new Ctor();`

1. 创建一个空对象 let obj = {}, 将 obj 的 `__proto__` 指向构造函数 Ctor 的 prototype `Object.setPrototypeOf(obj, Ctor.prototype)`
2. 将 obj 作为 this 执行构造函数体
3. 如果构造函数 Ctor 显式返回一个对象 (或函数), 则 new 的结果是该返回值, 否则 new 的结果是构造的 obj (显式返回一个原始类型会被忽略)

- 箭头函数没有 prototype, 也, 不能作为构造函数, new 调用箭头函数时会抛出 TypeError 错误
- new.target: 普通调用时为 undefined: `new.target === undefined`, new 调用时为构造函数: `new.target === Ctor`
-

js 模拟抽象类

```js
class Shape {
  constructor() {
    if (new.target === Shape) {
      throw new Error("Shape is abstract");
    }
  }

  area() {
    throw new Error("area() must be implemented");
  }
}

class Circle extends Shape {
  constructor(radius) {
    super(); // new.target === Circle
    this.radius = radius;
  }

  area() {
    return Math.PI * this.radius ** 2;
  }
}

new Shape(); // Error: Shape is abstract
new Circle(5);
```

### instanceof 的语义

A instanceof B 的语义: 在 A 的原型链上查找是否存在 B.prototype

```js
function instanceOf(maybeObj, maybeCtor) {
  if (
    maybeObj === null ||
    (typeof maybeObj !== "object" && typeof maybeObj !== "function") ||
    typeof maybeCtor !== "function"
  ) {
    return false;
  }

  let proto = Object.getPrototypeOf(maybeObj);
  while (proto !== null) {
    if (proto === maybeCtor.prototype) {
      return true;
    }
    proto = Object.getPrototypeOf(proto);
  }
  return false;
}
```

### 箭头函数

| 箭头函数没有 | 说明                                                         |
| ------------ | ------------------------------------------------------------ |

```ts
// GeneratorFunction `function* ()` 生成器函数
function* genFn() {
  yield 1;
  yield 2;
  return 3;
}

const gen = genFn(); // Generator 生成器对象

type IteratorResult<TYield, TReturn> =
  { done?: false; value: TYield } | { done: true; value: TReturn };

interface Generator<T, TReturn = unknown, TNext = unknown> {
  next(...value: [] | [TNext]): IteratorResult<T, TReturn>;
  return(value: TReturn): IteratorResult<T, TReturn>;
  throw(e: any): IteratorResult<T, TReturn>;
  [Symbol.iterator](): Generator<T, TReturn, TNext>; // Generator 生成器对象可以 for...of 遍历
}
```

### apply/bind/call

bind 后再 bind: bind 后 this 不改变

```js
function printA(...args) {
  console.log({
    thisA: this.a,
    arguments,
    args,
  });
}
const foo = { a: 1 };
const bar = { a: 2 };

let bound = printA.bind(foo, 3, 4);
bound = bound.bind(bar, 4, 5);

// thisA: 1
// arguments: [3, 4, 4, 5, 6, 7]
// args: [3, 4, 4, 5, 6, 7]
bound(6, 7);
```
